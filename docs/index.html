Introduction to Gedagadegadagadacode

Gedagadegadagadacode (which will be shortened to "GCode" from now on) is the primary coding language used to interact with Gedagadegadoperating System (which will be shortened to "GDOS" from now on) and is used both when navigating the system using the terminal and when creating programs that GDOS can run. A GCode program is usually comprised of a set of commands that are executed from top-to-bottom. GCode is unique in many ways, and although it does not have a real-world parallel, it still revolves around many real-life coding concepts such as variables, functions, and, to some extent, objects. In order to use GDOS, you must have some understanding of GCode.

Mathematical Expressions

Like most modern coding languages, GCode features built-in operators for mathematical expressions. These include:

  • The + operator, which adds two numbers.
  • The - operator, which subtracts the left number from the right one.
  • The * operator, which multiplies two numbers.
  • The / operator, which divides the left number by the right one.
  • The ^ operator, which raises the left number to the power of the right one.
  • The √ operator, which returns the left number's root of the right number. This symbol can be typed by pressing the down arrow on your keyboard.

GCODE DOES NOT USE STANDARD ORDER OF OPERATIONS! Instead, GCode performs mathematical operations from left-to-right. For example, 2+2*2 will return 8, not 6, because the interpereter will solve 2+2 and then multiply the result by 2.

Along with mathematical operators, GCode features some logical operators as well. These include:

  • The ? operator, which checks if two values are equal. This works with all data types.
  • The ! operator, which checks if two values are not equal. This also works with all data types.
  • The > operator, which checks if the left number is greater than the right one.
  • The < operator, which checks if the left number is less than the right one.

You may notice that when you try typing "true" or "false" into the terminal, the words get replaced by ⊤ or ⊥ symbols. These respectively represent the boolean values "true" and "false".

Examples

  • print: 1+1; - prints "2" in the terminal.
  • print: 3√27; - prints the 3rd root of 27, which is 3, in the terminal
  • print: 3√27+3/3+1^2; - performs the operations from left-to-right and prints "9" in the terminal.

    • Commands

    A GCode program is almost entirely comprised of commands. A command typically starts with a root command that is followed by sub-commands, each seperated by colons, and parameters that are also seperated by colons. All sub-commands and parameters must be seperated by colons, and every command line ends with a semicolon. Below are all of the built-in commands that GCode has to offer.

    print: [value];

    This command prints the given value to the terminal.

    help;

    This command brings you to this documentation page.

    memory: [sub-command, possible parameters];

    This command allows the program to store data in the program's runtime memory, acting very similar to variables in other programming languages. Runtime memory is reset every time a program runs, meaning keys stored here cannot be accessed once the program is stopped or restarted.
    The memory command must be followed by one of its sub-commands. Below is a list of its sub-commands and their uses.

    • memory: setkey: [keyname]: [keyvalue]; - Sets a key in runtime memory with the name given in the first parameter and the value given in the second. This command can be used to create new keys as well as update the values of existing ones.
    • memory: setarray: [keyname]: [keyvalues]; - Sets a key in runtime memory as an array with the name given in the first perameter and the specified values given in the further ones. Array values are seperated by colons.
    • memory: deletekey: [keyname]; - Deletes a key from runtime memory.
    • memory: clearall; - Clears runtime memory, deleting all data stored in it.
    • memory: listkeys; - Lists all keys currently stored in runtime memory and their respective values.

    terminal: [sub-command];

    This command can be used to write to the system's terminal. This command must follow its sub-commands, as listed below:

    • terminal: clear; clears all text in the terminal.

    regress: [numeric parameter]: unless: [conditional parameter];

    This command regresses the program's flow by an amount of lines determined by its first parameter, unless the second parameter is true. This is the intended way of creating loops in GCode.

    progress: [numeric parameter]: unless: [conditional parameter];

    This command progresses the program's flow by an amount of lines determined by its first parameter, unless the second parameter is true. This is the GCode equivalent of an "if" statement.
