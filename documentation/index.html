Introduction

If you've made it to this website, you probably have some idea of what you're gettinig yourself into.
Gedagadegadoperating System is a for-fun coding language / operating system built using JavaScript. Its very simple, with little complexity built-in to the coding language. However, trying to make something actually coherent is a challenge as, while it is technically possible to compute almost anything, it is wildly inefficient most of the time.
Here, I'll go over how to use GDOS (GeDagadegadOperating System) as well as how to code programs in GCODE (Gedagadegadagedacode)

Navigation

Unlike most modern 21st-century "operating systems", GDOS gives you zero built-in GUI for navigation. Instead, userrs must use terminal commands to navigate the system, although there really isn't much to "navigate".

Terminal

Everything you do in GDOS is done through the terminal. It is the primary interface for interacting with the system. In the terminal, you can run GCODE, which allows you to store or remove data from your permanent storage, create and run programs, and execute almost any other GCODE commands.

Permanent Storage

Your Permanent Storage is a place where you can store data and retrieve it between sessions. It works as a simple key-value system, where each key corresponds to a value that you can define. Use the save command to store data, with the first argument being the key and the second argument being the value. Whevever you want to retrieve data, type the key name for the data you want with a . character before it

Example:
save: name: "John Doe";
print: .name;

If this code doesn't make sense yet, don't worry - it will become clearer as you learn about GCODE's syntax.

GCODE

GCODE, formally Gedagadegadagedacode, is the programming language used for programming and navigation in GDOS. It is a relatively simple language that is focused on line-by-line command execution instead of object-oriented programming.

Syntax

In GCODE, each line represents a single command. Commands start with their name followed by a colon and arguments also separated by colons. Here's an example:

print: "Hello, World!";

In this example, print is the command name, and "Hello, World!" is the argument. The semicolon at the end of the line indicates the end of the command. Semicolons are extremely important in GCODE as they indicate the end of a line. Without them, only the first command in a program will execute, as every other command will be treated as an extraneous argument.
