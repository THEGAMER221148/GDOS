Introduction to Gedagadegadagadacode

Gedagadegadagadacode (which will be shortened to "GCode" from now on) is the primary coding language used to interact with Gedagadegadoperating System (which will be shortened to "GDOS" from now on) and is used both when navigating the system using the terminal and when creating programs that GDOS can run. A GCode program is usually comprised of a set of commands that are executed from top-to-bottom. GCode is unique in many ways, and although it does not have a real-world parallel, it still revolves around many real-life coding concepts such as variables, functions, and, to some extent, objects. In order to use GDOS, you must have some understanding of GCode.

Variables

Variables are stored in the system's temporary memory. The temporary memory can be accessed using the /mem commands.

Syntax

GCode's compiler reads variable definitions by checking for tildes (~). If a new line starts with that, the compiler reads the following code as a variable definition.

After initiating a variable definition, write the name of the variable. The variable's name must follow Javascript's property-naming conventions. This means that the variable's name:

  1. Cannot start with a number
  2. Cannot contain `, ', ", {, }, [, ], (, ), ., ,, ;, :, \, /, +, *, or any other special Javascript characters.
  3. Cannot contain spaces.
  4. Cannot be a Javascript keyword (document, window, String, etc.).

After writing the variable's name, add a colon (:) to tell the compiler that the name has ended and the next bit of code is the variable's definition.

In GCode, there are two types of variables: numbers and strings. To define a variable as a number, enclose the number in a pair of hashtags (##) and add a semicolon at the end. To define a variable as a string, enclose the message in graves (``) and add a semicolon at the end.

Examples

  • ~myVariable:#16#;
  • ~ myVariable: #16#;
  • ~message:`Hello, World!`;

Commands

In GCode, commands are indicated by a slash (/) character followed by the command's name and a colon. All commands must start with this format, and must end with a semicolon (;). After the first colon, additional sub-commands are added and are each seperated by colons. For example: /command:subcmd:subcmd2;.

GCode comes with many pre-defined commands. Here is a run-down of all of them and what they do.

Flow

Help

The /help:; command brings you to this page.

Mem

The /mem: command allows users to read and manipulate the system's temporary memory. The command has four sub-commands, as listed here:

  • /mem:add: adds data to temporary memory. Variable definitions should follow the colon, being seperated by tildes (~). For example: /mem: add: ~variable1: #5# ~variable2: `foo`;
  • /mem:clear:; clears all temporary memory.
  • /mem:help:; prints a small informational message about these commands.
  • /mem:list:; lists all of the top-level properties of the temporary memory object. It will not list further descendants.

Terminal

The /terminal: command's sub-commands all manipulate the terminal.

  • /terminal:clear:; clears all text in the terminal.
  • /terminal:print: prints the value leading the print sub-command. This can be a number, string, or variable value.

Flow

The /flow: command's sub-commands all control the flow of the program. They can be used to move back lines and skip lines.
